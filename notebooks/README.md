# Notebooks

Dieses Projekt verwendet bewusst `src/targeting_analysis.py` als reproduzierbaren Hauptworkflow.

Falls fuer die Gruppenpraesentation ein exploratives Notebook gewuenscht ist, kann auf Basis dieses Skripts ein Notebook `01_targeting_analysis.ipynb` erstellt werden. Die fachliche Logik bleibt identisch:

- Daten aus `mart.v_quartier_scoring_input` laden
- Explorationsschritte dokumentieren
- Score berechnen
- Top-Quartiere interpretieren

Fuer die Abgabe ist das Python-Skript ausreichend, weil es die Analyse klarer reproduzierbar macht.
